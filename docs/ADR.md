# ADR: Коллаборативный редактор для учебной документации

**Title**: Выбор платформы для совместного написания отчетов по командным дисциплинам  
**Decision**: Выбран сервис Etherpad  
**Дата**: 01.05.2026
**Авторы**: Фадеев Павел, Безуглов Денис, Никитин Александр

---

## Background

При выполнении командных учебных проектов студенты совместно готовят:

- Технические отчеты и пояснительные записки
- Документацию к лабораторным работам
- Описания архитектурных решений (ADR)
- Промежуточные и финальные презентационные материалы

Возникают следующие проблемы:

1. **Студент:**
   - Правки теряются при передаче файлов через мессенджеры
   - Конфликты версий при параллельном редактировании
   - Нет наглядности — кто что написал в документе
2. **Преподаватель:**
   - Невозможно оценить вклад каждого участника команды
   - Документы приходят в разных форматах и верстке
3. **Организация:**
   - Нежелателен внешний SaaS с хранением данных за рубежом
   - Нужно бесплатное и легко разворачиваемое решение

## Decision Outcome

Выбран **Etherpad** по следующим критериям:

| Критерий                    | Почему Etherpad                                                                                                           |
|-----------------------------|---------------------------------------------------------------------------------------------------------------------------|
| Прозрачность вклада         | Каждому участнику назначается свой цвет — преподаватель сразу видит, кто написал каждый фрагмент документа.              |
| Простота развертывания      | Не требует сложной инфраструктуры.                                  |
| Нулевой порог для студентов | Не нужна регистрация и установка — достаточно ссылки на пад в браузере.                                                  |
| Хранение в инфраструктуре вуза | Данные не покидают контур учебного заведения, нет зависимости от зарубежных облаков.                                  |
| История через Timeslider    | Встроенный инструмент позволяет «перемотать» документ и увидеть процесс его написания — полезно при разборе работы.      |
| Интеграция с Coder          | Ссылку на пад можно вставить прямо в README рабочего пространства студента, связывая код и документацию.                 |

## Consequences

1. **Для студентов:**
   - Исчезает проблема конфликтующих версий и потерянных правок
   - Каждый видит в реальном времени, что пишут коллеги по команде

2. **Для преподавателей:**
   - Объективная оценка вклада каждого участника без опроса команды
   - Единый формат сдачи: ссылка на пад + экспорт в .docx / .pdf

3. **Для учебного процесса:**
   - Документация создается параллельно с разработкой, а не в последнюю ночь
   - Простота инструмента не отвлекает от содержания работы

4. **Ограничения:**
   - Требуется резервное копирование базы данных падов
